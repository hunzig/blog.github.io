(原创)树莓派搭建免流服务器

Aug 25, 2016 | linux | Hits

根据网上教程自己全程搭建，网上教程鱼龙混杂，只有自己做过才知道对错

思路：openvpn + squid代理 +客户端免流配置文件 从而达到免流效果

实际上根据后期我在华硕路由器上搭建的经验来看，哪怕squid代理这一步不做，应该是照样也可以免流(无非就是不太稳定而已)

这里要注意的是如果用了squid，对应的端口必须是80 或者8080 ，而且需要做映射。我一开始没有用8080端口，导致免流失败。

本教程共需要映射两个端口，openvpn的1194端口和squid的8080端口

4.生成密钥

a.生成ca密钥 证书密钥

所有输入都默认就可以了，直接回车;keys文件夹下多了ca.crt ca.key

b.生成服务器密钥

接下来的输入中，common name 输入 server。其余默认或者打y，多出来三个文件 server.csr(证书签发请求文件) server.crt(公钥) server.key(私钥)

c.生成客户端密钥

enter pem pass phrase 输入客户端密钥的密码，输入两次。其他默认就好，又多了三个文件 client1.csr client1.crt client1.key
重复这一步可以生成用户二 client2

d.生成dh

这一步要等久一点，现在又多了一个文件 dh1024.pem
到现在为止，使用easy-rsa生成密钥部分，就完成了。

7.防火墙添加规则 （根据网上教程做的，不知道是否为必须）

#!/bin/sh

#允许接收目的端口为1194的包；1194就是server.conf中配置的端口
iptables -A INPUT -p udp –dport 1194 -j ACCEPT

#源地址转换：将vpn子网的ip替换为pi的本地ip
iptables -t nat -A POSTROUTING -s 10.8.0.0/24 -o eth0 -j SNAT –to-source 192.168.1.111
